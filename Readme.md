# Stock Price Forecaster

A time series forecasting system that predicts Reliance Industries (NSE) stock prices using a multi-feature PyTorch LSTM — compared against an ARIMA statistical baseline using walk-forward validation.

## What this project does

- Fetches **6 years of real NSE stock data** (2018–2024) via yfinance — no manual downloads
- Engineers 7 features from raw OHLCV data: Close, High, Low, Volume, daily return, H-L range, 7-day and 21-day moving averages
- Trains a **2-layer LSTM** on sliding windows of 30 trading days to predict the next day's closing price
- Runs an **ARIMA(5,1,0) walk-forward baseline** — refitting daily on expanding history to simulate real-world conditions
- **LSTM beats ARIMA** on all three metrics across 346 test days
- Forecasts the **next 30 trading days** autoregressively from the most recent data

## Feature engineering

Raw OHLCV data is expanded into 7 features before feeding to the LSTM:

| Feature | Description | Why it helps |
|---|---|---|
| Close | Closing price | Primary prediction target |
| High | Daily high | Upper price bound context |
| Low | Daily low | Lower price bound context |
| Volume | Shares traded | Market participation signal |
| Return | Daily % change | Momentum signal |
| HL_Range | High − Low | Volatility proxy |
| MA7 | 7-day moving average | Short-term trend |
| MA21 | 21-day moving average | Medium-term trend |

All features are scaled independently with `MinMaxScaler` before training.
Predictions are inverse-transformed back to INR before metric computation.

## Model — 2-layer LSTM

```
Input: (batch, 30, 7)        ← 30 trading days, 7 features
  → LSTM layer 1 (hidden=128, dropout=0.2)
  → LSTM layer 2 (hidden=128, dropout=0.2)
  → Last timestep output: (batch, 128)
  → Dropout(0.2)
  → Linear(128 → 1)
Output: (batch, 1)           ← next day closing price (scaled)
```

**Sliding window:**
```
Day 1–30   → predict Day 31
Day 2–31   → predict Day 32
Day 3–32   → predict Day 33
...
```

Each training sample is 30 consecutive trading days. The model sees
6 weeks of market history to predict the next day.

## Training

| Hyperparameter | Value |
|---|---|
| Optimiser | Adam (lr=0.001) |
| Loss | MSELoss |
| Scheduler | ReduceLROnPlateau (patience=5) |
| Early stopping | patience=10 |
| Gradient clipping | max_norm=1.0 |
| Batch size | 32 |
| Max epochs | 100 |
| Actual epochs | 12 (early stopping) |

**Why gradient clipping?**
LSTMs backpropagate through 30 timesteps. Without clipping, gradients
can grow exponentially — `clip_grad_norm_(max_norm=1.0)` caps magnitude
without changing direction.

**Why early stopping at epoch 12?**
Val loss plateaued — the model converged fast with multi-feature input.
Continuing would overfit to training noise.

## ARIMA baseline — walk-forward validation

ARIMA(5,1,0) is run in walk-forward mode: predict one day, observe
actual price, add to history, refit, repeat — across all 346 test days.
This simulates real-world daily forecasting accurately.

**Why walk-forward and not a single fit?**
Fitting ARIMA once and predicting 346 days ahead degrades rapidly.
Walk-forward matches how the model would actually be used in production.

## Results

Evaluated on 346 test days (20% holdout, chronological split).

| Metric | LSTM | ARIMA | Winner |
|---|---|---|---|
| RMSE | **18.02** | 18.11 | LSTM |
| MAE | **12.63** | 12.74 | LSTM |
| MAPE | **0.93%** | 0.94% | LSTM |

**LSTM wins on all three metrics.**

0.93% MAPE means the model's average prediction error is less than
₹14 on a ~₹1,400 stock — across 346 trading days of live NSE data.

The margin over ARIMA is intentionally small — stock prices are close
to a random walk, so a consistent edge over a statistical baseline is
the realistic and meaningful result. LSTM's advantage comes from
incorporating volume, volatility, and moving averages that ARIMA
cannot use.

## Key concepts demonstrated

- **Sliding window** — converting a price sequence into supervised ML samples
- **Temporal split** — no shuffling; chronological order preserved to prevent data leakage
- **MinMaxScaler** — why scale + how to inverse-transform multi-feature predictions
- **LSTM gates** — forget, input, output gates learning which history to retain
- **Gradient clipping** — preventing exploding gradients across 30 backprop timesteps
- **Early stopping** — halting when val loss stops improving on noisy financial data
- **Walk-forward validation** — realistic time series evaluation methodology
- **ARIMA baseline** — why classical statistical models are essential comparison points
- **Autoregressive forecasting** — using predictions as inputs for future predictions
